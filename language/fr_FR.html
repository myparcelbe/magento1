Introduction

Ce manuel décrit les fonctionnalités de l’extension MyParcel BE et vous explique comment l’installer dans votre boutique en ligne.

Nous expliquons l’installation via SSH/FTP dans ce manuel. Si vous avez besoin d’aide, nous vous recommandons de contacter votre gestionnaire technique.

Cette extension Magento vous permet de créer deux types d’envois MyParcel BE:

  1. Normal (colis classiques et collecte à un point bpost)
  2. Colis vers l’étranger, dans l’UE et hors UE.

Sommaire

1 Installation

Dans tous les cas, nous recommandons d’abord de procéder à une installation dans un environnement de test et d’effectuer une sauvegarde de votre boutique en ligne.

1.1 Exigences techniques

Pour utiliser l'extension MyParcel BE Magento, votre serveur doit répondre aux exigences suivantes:

  • Version de PHP 5.3 minimum
  • Bibliothèques SOAP PHP
  • 256 Mo de mémoire

Les versions de Magento suivantes sont supportées: 
 Community 1.6, 1.7, 1.8, 1.9

1.2 Télécharger l’extension

L’extension peut être téléchargée via le lien suivant:

  • Github MyParcel pour la version la plus récente (cliquez sur source code sous Téléchargements pour télécharger le fichier zip).

1.3 Désactiver le compilateur

Avant de commencer l’installation, il est important que le compilateur soit désactivé (s'il est activé) dans votre installation Magento. Une fois connecté à votre environnement Magento, vous pouvez désactiver le compilateur via le menu: Système → Outils → Compilation.

1.4 Télécharger des dossiers et des fichiers

Le fichier .zip contient deux dossiers:

  • app
  • skin

Téléchargez ces dossiers vers votre installation Magento. 
Aucun fichier ne sera écrasé. Si le programme SSH / FTP vous demande si vous souhaitez fusionner ou remplacer les dossiers, sélectionnez l'option fusionner.

1.5 Vider le cache et se reconnecter

Connectez-vous à votre environnement de gestion Magento et choisissez le menu Système → Gérer le cache. Cliquez sur Sélectionner tout, choisissez l’option ‘Rafraîchir’ dans le menu déroulant et cliquez sur ‘Confirmer’. 
Cliquez ensuite sur ‘Se déconnecter’ dans le coin supérieur droit, puis reconnectez-vous à votre environnement de gestion.

1.6 Vérification cron

L’extension MyParcel fait appel à certaines tâches cron. Celles-ci sont nécessaires afin d’adapter périodiquement les statuts des envois dans votre back-end. L’extension se fonde sur la fonctionnalité cron de Magento. 
Prenez contact avec votre développeur en cas de question à ce sujet.  

Serveur tâches cron
La manière de paramétrer les tâches cron sur votre serveur diffère selon le fournisseur d’hébergement, car il existe plusieurs systèmes de gestion de serveur web. En voici quelques exemples : DirectAdmin, Plesk et Cpanel. Vous trouverez ci-dessous un exemple de configuration d’une tâche cron pour Magento dans DirectAdmin.

  1. Accédez au compte sur lequel l’environnement Magento est en cours d’exécution et cliquez sur Cronjobs.
  2. Pour une boutique en ligne Magento Community, paramétrez que la tâche cron doit être activée toutes les 5 minutes (comme dans l’exemple ci-dessous).
  3. Vérifiez avec SSH si vous pouvez activer la tâche cron.
    Cela peut se faire en se connectant à une fenêtre du terminal et en entrant la commande. Dans notre exemple, il s’agit de /home/magento/public_html/cron.sh. Vous devrez le faire avec les droits ‘root’. Si cela ne fonctionne pas ou si le serveur ne dispose pas d’une option ‘shell’, vous pouvez alors basculer vers le fichier cron.php. Dans ce cas, vous pouvez créer deux tâches cron. Vous pouvez préciser où PHP tourne pour les requêtes et localiser l’emplacement avec la constante PHP PHP_BINDIR. 
    Vous devez placer un paramètre derrière les requêtes :
    • Requête 1: cron.php -malways 1
    • Requête 2: cron.php -mdefault 1
    La capture d'écran ci-dessous vous fournit un exemple.
  4. Assurez-vous que le fichier cron.sh (ou cron.php) dispose de 755 droits. Le serveur appelle maintenant la fonctionnalité cron de Magento, toutes les 5 minutes.
  5. Magento cron
    1. Allez dans le menu Système → Configuration → Avancé → Système → Cron (Tâches planifiées) .
    2. Assurez-vous que les paramètres sont les mêmes que ceux repris dans l’image ci-dessous.
    Il est préférable d’ouvrir la base de données et d’ouvrir la table cron_schedule pour vérifier que le cron s'exécute correctement. Il sera directement complété. Vous pourrez ainsi vérifier que les modifications sont bien effectives.

2 Configuration

Allez dans le menu Système → Configuration → Ventes → MyParcel pour configurer l’extension MyParcel. 
Sous la rubrique ‘Version et Support’, vous pouvez voir quelle est la version de l’extension que vous utilisez.

Vous aurez également le choix entre 2 onglets. Un onglet pour les paramètres généraux et un autre pour les paramètres de paiement.

2.1 Paramètres généraux

Dans l’onglet ‘Paramètres généraux’, vous pourrez modifier les ‘Paramètres API’, les ‘Paramètres de base’ et les ‘Paramètres d’expédition standards’, et / ou les ajuster selon vos souhaits.

2.1.1 Paramètres API

Vous pouvez introduire ici votre nom d’utilisateur API et votre clé API. Ceux-ci sont nécessaires au transfert des envois vers MyParcel. 
Ces données peuvent être trouvées dans votre compte MyParcel BE sur backoffice.sendmyparcel.be en choisissant Paramètres -> Général

2.2 Paramètres de base

Cette rubrique vous permet de saisir tous les paramètres relatifs à l’utilisation de l’extension MyParcel BE.

2.2.1 Orientation de l’impression

Sélectionnez orientation d'impression. Si vous choisissez le format A4, quatre étiquettes d’expédition seront imprimées par feuille A4. Si vous choisissez le format A6, vous aurez besoin d’étiquettes d’expédition qui sont compatibles avec une imprimante à étiquettes spéciales comme une Zebra ou une Dymo 4XL.

2.2.2 Modèle d’e-mail Track & Trace

Pour l’envoi des e-mails Track & Trace, vous avez le choix entre 2 options. Vous pouvez opter :

  1. pour l’envoi des e-mails Track & Trace via MyParcel BE ou
  2. pour l’envoi de ceux-ci via votre boutique Magento.

Via MyParcel BE

Suivez les étapes ci-dessous:

  1. Connectez-vous sur sendmyparcel.be
  2. Allez dans le menu Paramètres → Track & Trace
  3. Paramétrez vos e-mails Track & Trace selon vos souhaits.

Via votre boutique en ligne Magento:

Suivez les étapes ci-dessous:

  1. Allez dans Système → E-mails transactionnels
  2. Cliquez sur le bouton Ajouter un nouveau modèle
  3. Sélectionnez le modèle Mise à jour de l’envoi et chargez le modèle.
  4. Donnez un nom au modèle dans le champ Nom du modèle. Ceci est pour un usage interne.
  5. Adaptez le contenu du modèle selon votre souhait. Pour ce faire, vous pouvez utiliser les variables {{var tracktrace_url}} et {{var retourlabel_url}}.
  6. Pour vous assurer que seuls les textes sont affichés lorsque ces variables sont présentes, vous pouvez utiliser {{depend tracktrace_url}} et {{depend retourlabel_url}}.
  7. Cliquez sur le bouton Enregistrer le modèle.
  8. Allez dans le menu Système → Configuration → MyParcel et ensuite dans l’onglet Paramètre généraux.
  9. Dans le menu déroulant du modèle d’e-mail Track & Trace sélectionnez le modèle que vous venez de créer.

2.2.3 Utiliser MyParcel pour tous les modes d’expédition

Choisissez l’option Oui pour transférer et utiliser dans MyParcel BE tous les modes d’expédition existants dans Magento.
Si vous ne souhaitez pas utiliser cette option, choisissez Non. Vous pouvez maintenant choisir vous-même les modes d’expédition qui peuvent être utilisés par le plug-in MyParcel BE.

2.3 Paramètres d'expédition standards

2.3.1 Options d’expédition standards

Les options suivantes sont activées en fonction de la valeur de la commande. Exemple : si vous saisissez la valeur 100 dans Assurance jusqu’à 500 €, les commandes d'une valeur d’achat supérieure à 100 € seront automatiquement assurées à concurrence de 500 €. Cette logique s’applique également pour le paramétrage de laSignature pour réception.

Attention : l’option n’est pas activée si elle est à0 L’option est active si une valeur de 1 ou supérieure est saisie.

2.3.2 Type d’envoi douanier

Choisissez l’option qui s’applique à vos envois dans le menu déroulant du champ de configuration Type d'envoi douanier.
Il s’agit d’un paramètre standard pour les envois ‘Reste du monde’. Lorsque vous transférez un envoi vers MyParcel BE, vous pouvez toujours adapter cet élément pour chaque envoi. Cette option ne s’applique pas si les étiquettes sont créées par lot.

Code HS des marchandises en douane

Introduisez le code HS souhaité dans le champ Code HS des marchandises en douane. 

Les codes sont disponibles via ce lien: https://financien.belgium.be/nl/douane_accijnzen
Attention : le code HS doit comporter quatre chiffres exactement. 

Il s’agit d'un paramètre standard pour les envois ‘Reste du monde’. Lorsque vous transférez un envoi vers MyParcel BE, vous pouvez toujours adapter cet élément pour chaque envoi. Cette option ne s’applique pas si les étiquettes sont créées par lot. 

Il est également possible de donner un code HS propre à chaque catégorie de produits. Cette option peut être complétée au bas de la page de la catégorie souhaitée. Allez dans Catalogue -> Gérer les catégories..

2.4 Configuration des modes d’envoi

Allez dans le menu Système → Configuration → Vente → Modes d’envoi → MyParcel.
Activez le mode d’envoi. Choisissez ensuite un titre et un nom de mode.

2.4.1 Type de tarif

Deux méthodes peuvent être utilisées pour calculer les frais d’expédition:

  • Méthode simple : avec cette méthode, vous saisissez directement le type et le prix.
  • Méthode par tableau : vous utilisez les ‘Table Rates’ de Magento.
    1. Allez dans le menu Système → Configuration → Modes d'expédition → Myparcel.
    2. Choisissez le Type de tarif pour Table et paramétrez Conditions elon votre souhait.
    3. Désactivez vous-même le tableau des tarifs du mode d’expédition de Magento. Pour désactiver le mode d’expédition de Magento, rendez-vous sur Mode d’expédition → Tableau des prix tarifs et choisissez Non dans Activé.
    4. Créez un fichier CSV Table Rates. Cette page vous permet de générer un fichier CSV: www.elgentos.nl/tablerates.
      Astuce: pour fixer la gratuité des frais d’expédition à partir d'un certain montant, créez une ligne supplémentaire au niveau du pays concerné.
    5. Veillez à travailler au niveau du site internet. Sélectionnez ici le site internet principal.
    6. Téléchargez le CSV Table Rates dans le mode d'expédition Table Rates. L’extension MyParcel BE utilise les Table Rates de Magento.

3 Paramètres de checkout

Sous l’onglet ‘Paramètres de checkout’, vous pouvez appliquer tous les paramètres qui seront visibles au moment du checkout. Cet onglet est divisé en ‘Paramètres de base’, ‘Livraison’ et ‘Collecte à un point bpost’.

3.1 Paramètres de base

3.1.1 Nombre de jours

La fonction ‘nombre de jours à l'avance’ vous permet de définir combien de jours à l’avance votre client peut choisir sa livraison. Lorsque vous optez pour 7 jours, la dernière possibilité de choix sera donc exactement 7 jours plus tard. Les jours durant lesquels aucune livraison n’est possible ne sont pas affichés lors du checkout.

Dans l’image ci-dessus, vous pouvez voir qu’il a été choisi d’afficher 7 jours à l'avance. L’image ci-dessous vous présente ce que le client peut voir lors du checkout. Un total de 5 jours de livraison est affiché car le dimanche et le lundi ne sont pas des jours de livraison. Le dernier moment de livraison est exactement 7 jours plus tard.

3.1.2 Dernier moment de commande

Vous pouvez définir ici jusqu’à quel moment il vous est possible de préparer une commande pour la livraison. Si vous saisissez 16:00, la première possibilité de livraison sera portée à un jour ultérieur pour tous les clients qui passeront commande après cette heure.
Par exemple : votre délai de traitement est d’un jour et vous pouvez traiter les commandes jusqu’à 16h00. Pour une commande reçue le mardi 4 octobre à 16h02, la première possibilité de livraison pour votre client sera donc le jeudi 6 octobre. Cela signifie que vous devrez déposer votre colis le mercredi à un point bpost.

3.1.3 Jours d’envoi

Dans ‘jours d'expédition’, vous pouvez indiquer les jours lors desquels il vous est possible de fournir des colis pour un enlèvement le jour même. Au niveau du checkout, cet élément sera pris en compte pour les options proposées à votre client.

3.1.4 Retard

Vous pouvez indiquer ici si vous avez besoin d’un délai de traitement plus long pour procéder à la livraison. Au niveau du checkout, cet élément sera pris en compte pour les jours de livraison possibles proposés à votre client.

3.1.5 Couleurs

Il est possible d’adapter la couleur de la fenêtre dans laquelle les options d’expédition sont affichées selon votre charte graphique.

3.2 Livraison

3.2.1 Titre livraison

Vous pouvez choisir le titre que vous donnez à la livraison. Nous vous conseillons d’indiquer ‘bpost’.

3.2.2 Signature pour réception

Vous pouvez indiquer si vos clients peuvent choisir cette option lors du checkout. Si vous souhaitez l’afficher, choisissez Oui Il est possible de renommer cette option ; dans l’exemple, elle s’appelle ‘Signature pour réception’. Vous pouvez également associer des frais complémentaires à cette option. Dans l’exemple cette option coûte 0,45 € supplémentaires.

Une option a été également créée pour un supplément combiné des choixLivrer uniquement chez moi et Signature pour réception. Vous pouvez remplir un prix à votre discrétion.

3.2.3 Livraison le samedi

Vous pouvez également choisir de faire livrer un colis le samedi. N'oubliez pas de sélectionner le vendredi comme jour d’envoi.

3.3 Collecte dans un point bpost

Avec cette option, vos clients peuvent choisir d’aller chercher leur colis dans un point bpost. Ils peuvent choisir le point en question lors du checkout. Vous avez la possibilité de facturer un supplément pour ceci, même si MyParcel BE ne facture pas de frais supplémentaires.

4 Fonctionnalités back-end

Pour utiliser les fonctionnalités de back-end, l’extension MyParcel BE doit être activée.

4.1 Créer un envoi

Vous pouvez créer un envoi MyParcel BE de deux manières:

1. Méthode par lots via l’aperçu des commandes.

  1. Sélectionnez les commandes que vous voulez exporter vers MyParcel BE.
  2. Choisissez ‘Créer des étiquettes’ dans la barre d'actions MyParcel et cliquez ensuite sur ‘Envoyer’.
  3. Sélectionnez la position d'impression correcte si vous avez choisi ‘A4’.
  4. Le PDF apparaît sur votre ordinateur et est prêt à être imprimé.

2. Par commande, lorsque vous créez un envoi Magento.

  1. Cliquez sur la commande correspondante.
  2. Cliquez sur le bouton Envoyer en haut à droite.
  3. Un cadre jaune apparaît dans le coin inférieur droit. Vous pouvez ici adapter le type d’envoi, de même que les options d’expédition disponibles par type d’envoi.
  4. Transférer l’envoi vers MyParcel BE.

Il est également possible d'envoyer une commande via des livraisons partielles, par exemple lorsque la commande totale est trop lourde ou trop grande pour être envoyée dans un seul colis, ou qu’une partie de celle-ci est temporairement indisponible et que vous souhaitez l’envoyer ultérieurement.

  1. Vous pouvez afficher la commande à cet effet et choisir ‘Envoyer’ en haut à droite.
  2. Modifiez la quantité dans Articles à envoyer et sélectionnez Transférer l’envoi en bas à droite. .
  3. ous pouvez ensuite créer, de la même manière, une étiquette d’expédition pour la même commande. Les deux envois sont sauvegardés sous la même commande.

4.2 Générer une étiquette de retour

Dans l’aperçu des commandes, cliquez sur la commande correspondante. Sur l’écran de détails, cliquez sur l’onglet ‘Expéditions’ dans la partie gauche du menu. Cliquez ensuite sur l’envoi qui doit être retourné. Vous êtes maintenant sur l’écran de détail d'expédition, avec le bouton ‘Générer une étiquette de retour’ dans le coin supérieur droit.

Si une adresse e-mail a été renseignée, votre client recevra ensuite un e-mail de MyParcel BE contenant un lien vers l’étiquette d’expédition. Vous avez aussi la possibilité de personnaliser votre e-mail pour les retours dans MyParcel. Il est également possible de définir une contribution propre du client. Pour ce faire, connectez-vous à MyParcel BE et allez dans Paramètres → Retour , puis en bas de la page Personnaliser l’e-mail.

4.2.1 Générer une étiquette de retour non spécifiée

Vous pouvez envoyer une étiquette de retour à votre client depuis l’aperçu de vos commandes. Pour ce faire, cliquez sur le bouton Envoyer une étiquette de retour


Une fenêtre s’ouvrira ensuite pour vous permettre d’indiquer le nom et l'adresse e-mail de votre client. Il est également possible de fournir une référence pour identifier la commande.

6 Questions fréquemment posées

6.1 Mon système de checkout est-il supporté?

Le mode d’expédition MyParcel BE utilise les fonctionnalités standards de Magento. Cela signifie que la méthode d’expédition fonctionne avec chaque système de checkout qui a été développé selon les directives de Magento.
Les extensions de checkout suivantes sont compatibles avec la ‘Collecte à un point bpost’ :

  • OnePageCheckout, qui est le checkout standard de Magento.
  • IWD checkout
  • Idev OneStepCheckout, dqui peut être acheté via www.onestepcheckout.com. Attention : vous ne devez jamais définir ‘Collecte à un point PostNL’ comme mode d’expédition par défaut.
  • Fire Checkout (support depuis Fire Checkout).

6.2 J’utilise une extension de vérification du code postal, est-ce un problème?

En principe, il est possible d’utiliser une extension de vérification de code postal en combinaison avec l’extension MyParcel Magento.

6.3 Comment puis-je configurer la livraison gratuite?

A côté des Tables Rates, vous pouvez également définir des frais d’envoi gratuits via Promotions > Règles de prix du panier.

  1. Cliquez en haut à droite sur Ajouter une nouvelle règle.
  2. Donnez un nom à la règle dans Données de la règle (par ex.: ‘Envoi gratuit à partir de 50’).Activez le statut et sélectionner toutes les options dans sites internet et groupes de clients.
  3. En haut à gauche, cliquez sur Conditions et précisez vous-même la règle en cliquant sur +. (Par ex. ‘Sous-total est égal ou supérieur à 50’)
  4. A l’étape suivante, indiquez ‘Seulement pour les articles concernés’ dans ‘Expédition gratuite’.
  5. Cliquez sur ‘Enregistrer’ dans le coin supérieur droit.
